Bloccit

Overview

News, sports, music, movies, technology, science, food, travel and beyond - for all topics of conversation in our everyday lives, we have convenient sites such as Reddit that aggregate this information for us. I have created Bloccit, a Ruby on Rails application, to provides a similar experience. It is a forum space for users, and depending on their authorization level, users have the ability to create an account, create posts, and comment on/favorite other users posts.

I created this app while taking part in the Bloc Full Stack Web Development course.


See on GitHub

Features

  • Sessions were created to keep track of a users state, keeping them logged in to the site
  • User specific roles - Admin, Member, and Guest with authorization rules for each level were created
  • Users have the ability to upvote/downvote or favorite posts
  • Users have a profile page where they can view their posts and favorites
  • Posts are filtered by a time-decay algorithm and voting score

Ruby Gems

  • BCrypt - for sensitive user information encryption
  • Bootstrap - CSS framework
  • Figaro - to securely store Sendgrid and Devise credentials as environment variables
  • RSpec - testing framework for test driven development (TDD)

Bloc-Jams

Overview

Music plays a huge part in my life - from James Taylor to the Beatles, Coldplay to Drake, and even up-and-coming EDM artists like Martin Garrix to the famed Calvin Harris, I listen to it all. For my first app, I wanted to make a product I would love to use, and music made it an easy call. I created Bloc-Jams, which is a music player web-app similar to Spotify. Bloc-Jams was originally created using DOM scripting and jQuery library but later expanded using Angular for scalability and robustness.

My goal in creating Bloc-Jams was to learn how to configure Angular routing and states, implement controllers for my app's views, create a service that handles song playback, and write a directive that controls song and volume sliders.

I created this app while taking part in the Bloc Full Stack Web Development course.


See on GitHub

Features

  • The app features view templates for a landing page, album collection page, individual album page, and a player control bar
  • MVC architecture was utilized to dynamically build the views using controllers
  • Song playback was handled by injecting a service to share the song playback information across all components of the app
  • Directives were used to bind custom user actions to angular functionality, such as adjusting the playback time or volume

Wikibloc

Overview

It’s everyone’s quick and easy go-to source for information, that first Google search result you’re looking for: Wikipedia. In a more general sense - wikis - are the perfect way to collaborate on community sourced content. With Wikibloc, I have created my own app that allows users to create their own wikis and share them with collaborators.

I created this app while taking part in the Bloc Full Stack Web Development course.

See on GitHub

Features

  • Wikibloc offers different access levels by user role: admin, standard (free), or premium (subscription)
  • Standard users can create, read, update, and delete public wikis
  • Users can upgrade their account from standard to a paid account
  • Premium users have the ability to create private wikis
  • Wikis are editable using markdown syntax
  • Premium users can add or remove collaborators from their private wiki pages

  • Pundit - role authorization
  • Stripe - user account upgrades
